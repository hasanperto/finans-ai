FinansAI — Sistem Özeti, Mobil/PWA & Geliştirme Yol Haritası

Tarayıcı tabanlı kişisel finans paneli. Bu belge mevcut sistemi, son yapılan UI/UX + PWA + tablet/mobil iyileştirmelerini, tespit edilen hata/eksikleri ve önerilen geliştirme yol haritasını özetler. Tarih: 2026-05-23.

PWA (offline) Mobil + Tablet uyumlu TR / DE Next.js 16.2.4 · React 19 IndexedDB (Dexie) yerel veritabanı Şifre PBKDF2-SHA256 hash JSON import/export Bütçe modülü (kategori limitleri + uyarı)

1) Program Nedir?

  • Amaç: Gelir/gider kaydı, kategorilendirme, aylık özet ve grafiklerle takip; istenirse AI asistanla finans soruları.
  • Çalışma biçimi: Backend/DB zorunlu değil; veriler tarayıcıda saklanır.
  • Dil: Türkçe ve Almanca arayüz (mobilde de hızlı geçiş).
  • Para birimi: TRY/EUR/USD/GBP/CHF + (SILBER/GOLD) gösterim.
  • Cihazlar: Telefon, tablet, masaüstü — responsive + PWA destekli.

2) Kullanılan Teknoloji

  • Framework: Next.js 16.2.4 (App Router, Turbopack)
  • UI: React 19.2.4, TypeScript
  • Grafikler: Recharts
  • İkonlar: lucide-react
  • Lint: ESLint (Next.js config)
  • PWA: manifest.json + /sw.js service worker (static + runtime cache)
  • Yerel veritabanı: Dexie 4 üzerinden IndexedDB (offline, GB seviyesinde, indeksli).
  • Kriptografi: Web Crypto API — PBKDF2-SHA256 (100.000 iter, 16 byte salt).
  • Deploy: Netlify (build = npm run build, publish = .next)

2.5) Yerel Veritabanı & Kişisel Mod (offline)

Cihaz başına ayrı, tamamen offline çalışan kişisel kullanım için yeniden yapılandırıldı.

  • Veri katmanı: Artık localStorage değil IndexedDB (Dexie sarmalı). 3 store: transactions, recurring, meta. date / type / category / isActive / nextDue indeksli.
  • Otomatik göç: İlk açılışta localStorage'daki tüm veri IndexedDB'ye taşınır, finansai_db_migrated_v1 bayrağı kurulur. Eski anahtarlar silinmez (güvenli geri dönüş).
  • Şifre güvenliği: Düz metin saklama kaldırıldı; PBKDF2-SHA256 (100k iter + per-secret salt) hash. Format: pbkdf2$<iter>$<saltB64>$<hashB64>. Karşılaştırma sabit zamanlı.
  • Eski şifre yükseltme: Var olan düz metin şifre ilk doğru girişte otomatik hash'lenip kaydedilir — kullanıcı için fark yok.
  • Gizli soru yanıtı: Aynı hash yöntemiyle saklanıyor; "Şifremi unuttum" akışı bozulmadan çalışıyor.
  • JSON yedek: Hem "İndir" hem de "İçe aktar" akışı eklendi — başka cihaza taşıma artık tek dosyalık iş.
  • Tekrarlayan otomatik üretim: Yükleme + sekme görünür olduğunda + her saat tetikleniyor (catch-up, geçmiş günler için).
  • Kapasite: ~5 MB localStorage sınırı yerine pratik olarak GB seviyesinde IndexedDB; binlerce işlem performans sorunu yaratmaz.
  • Internet gerekmiyor: Veritabanı tamamen cihazda. Yalnızca kur servisi (frankfurter) ve AI çağrıları opsiyonel ağ — biri başarısızsa son değer/önbellek kullanılır.
PWA + Lokal DB Telefon ve tablet için kişiye özel tasarım: cihaz açılışta uygulamayı yükler, oradan sonra internet olmasa bile gelir/gider girişi, raporlar ve filtreler çalışır. Yedek "İndir"/"İçe aktar" ile cihazlar arası taşıma manuel olarak yapılabilir.

2.6) Bütçe Modülü (yeni)

Kategori başına aylık üst sınır + ilerleme + uyarılar. Telefonda en sık bakılacak görünüm.

  • Veri: Settings.budgets: Budget[] — her giriş { categoryId, amount, period: 'monthly' }. Mevcut meta store'unda settings içinde saklanır; ayrı store gerekmedi.
  • Hesap: src/app/lib/budget.ts — saf fonksiyon, içinde bulunulan ayın gider toplamını kategoriye göre çıkarıp her bütçe için { spent, remaining, percent, level } üretir. Seviye eşiği: %80 → warn, %100 → over.
  • Ayarlar UI: Yeni Bütçeler sekmesinde her gider kategorisi için input + Kaydet/Kaldır + canlı ilerleme çubuğu (yeşil/sarı/kırmızı).
  • Dashboard: En sıkışık 5 bütçe ilerleme çubuğu olarak gösterilir. Aşılan kategori varsa kırmızı banner ("{n} kategoride aylık bütçe aşıldı"), aşılma yoksa ama %80'i geçen varsa sarı uyarı banner'ı.
  • i18n: budget.* anahtarları TR/DE sözlüklere eklendi.
  • Erişilebilirlik: Bütçe çubukları, durum renklerinin yanına yüzde ve sayısal toplam — sadece renk üzerinden anlam vermiyor.

3) Modüller / Sayfalar

  • Dashboard (/): Toplam gelir/gider, net bakiye, tasarruf oranı, son 6 ay barı, kategori pasta grafiği, AI önerisi, son işlemler.
  • Gelirler (/income): Gelir ekleme/düzenleme/silme, tekrarlayan gelir, kategori dağılımı, yıllık projeksiyon.
  • Giderler (/expenses): Gider ekleme/düzenleme/silme, tekrarlayan gider, zorunlu/isteğe bağlı ayrımı.
  • İşlemler (/transactions): Tarih (bu ay, geçen ay, 3/6 ay, tümü), tür filtresi, arama, CSV dışa aktarma.
  • AI Asistan (/assistant): Gemini / OpenAI / MiniMax / OpenRouter / Custom sağlayıcı, finansal bağlam ile soru-cevap.
  • Ayarlar (/settings): Dil, para birimi, AI ayarları, güvenlik (gizli soru), kategoriler, bütçeler, raporlar, yedekleme (JSON) ve örnek senaryolar.

8) Önerilen geliştirme yol haritası

Faz 1 · Stabilite
  • JSON import (yedekten geri yükleme) — Settings içine dosya inputu.
  • Tekrarlayan işlem üretimi: visibility-change + her saat setInterval tetiği.
  • Örnek verilerin tarihlerini "bugüne göre" formata çekmek.
  • Şifreyi Web Crypto API (PBKDF2/SHA-256 + salt) ile hash'lemek.
  • i18n: kalan sabit Türkçe stringleri sözlüğe almak.
Faz 2 · Özellik
  • Bütçe modülü: Kategori bazında aylık limit + ilerleme çubuğu + aşıldığında dashboard banner.
  • Hesap (account) modeli: Banka / nakit / kredi kartı ayrı bakiyeler, hesap içi transfer.
  • Hedefler: Tasarruf hedefi (örn. 6 ayda 60.000 ₺) + ilerleme.
  • Ekstre içe aktarma: CSV / OFX / camt.053 parse (DE banka standardı) — manuel kayıttan kurtulma.
  • Daha zengin raporlar: Yıllık trend, kategori ısı haritası, yıl/yıl karşılaştırma.
Faz 3 · Çoklu cihaz / bulut
  • Supabase (veya Firebase) auth + Postgres senkron — telefon ↔ tablet ↔ web aynı veriyle çalışır.
  • Çakışma çözümü (last-write-wins veya CRDT — küçük veri için LWW yeterli).
  • Sunucu tarafı kur servisi cache'i (frankfurter çağrısı tek noktadan).
  • E2E şifreleme opsiyonu (zero-knowledge) — kullanıcı verisi sunucuda da şifreli kalır.
Faz 4 · AI & Otomasyon
  • İşlem otomatik kategorilendirme (kural + AI fallback).
  • Doğal dil ekleme: "dün markete 540 TL" → işlem.
  • Aylık AI raporu (mail/push olarak özet + öneri).
  • Anomali tespiti (geçen aya göre ani artışlar).
  • Sesli giriş (Web Speech API) — sürücü modu için.

9) Veri Yapısı ve Depolama

  • Birincil depolama: IndexedDB / finansai_db — store'lar: transactions, recurring, meta.
  • Kalan localStorage anahtarları: finansai_lang (dil), finansai_fx_rates_v1 (kur cache), finansai_unlocked_v1 (oturum), finansai_install_dismissed_v1, finansai_db_migrated_v1 (göç bayrağı).
  • Yedek: Ayarlar > Veri Yönetimi > JSON İndir ve İçe aktar.
  • FX cache: 6 saatten yeni ise yeniden çekilmez; offline'da son kur ile çalışır.
  • Veri formatı: Yedek JSON v2 — { version, exportDate, transactions, recurringTransactions, settings }.

10) Giriş / Güvenlik

  • Şifreli giriş opsiyoneldir: Ayarlar > Güvenlik > "Giriş Koruması".
  • Şifre saklama: PBKDF2-SHA256 (100.000 iter, 16 byte salt) hash; karşılaştırma sabit zamanlı.
  • Gizli soru yanıtı: Aynı yöntemle hash'li olarak saklanır.
  • Şifremi unuttum: Giriş ekranında gizli soru yanıtı doğrulanır → yeni şifre.
  • Oturum: Açılış kilidi sessionStorage üzerinden, sekme kapanınca yeniden istenir.
  • Eski veriye geri uyum: Düz metin saklanan eski şifre, ilk doğru girişte otomatik olarak hash'lenerek kaydedilir.
Yerel + hash Şifre cihazda hash olarak duruyor; ham şifre hiçbir yerde saklanmıyor. Çoklu cihaz/bulut senkron istenirse sunucu kimlik doğrulama (Faz 3) ayrıca planlanmalı.

11) Hızlı çalıştırma

  • npm install
  • npm run dev — geliştirme (Turbopack).
  • npm run build & npm start — prod.
  • npm run lint — eslint.
  • Lighthouse / DevTools > Application: Manifest, Service Worker, Installability başarılı.

12) Deploy hazırlığı (Netlify)

  • Plugin: @netlify/plugin-nextjs devDependency olarak eklendi; netlify.toml üzerinden register edildi.
  • netlify.toml: Hatalı /* → /index.html redirect kaldırıldı (App Router ile uyumsuzdu). Build command npm run build; publish dir plugin tarafından yönetildiği için manuel set edilmedi.
  • Cache-Control header'ları: /sw.js → max-age=0, must-revalidate + Service-Worker-Allowed: /. /manifest.json → 1 saat.
  • .gitignore: .netlify/ klasörü artık ignore listesinde.
  • README: Geliştirme + deploy + PWA notları sıfırdan yazıldı.
  • Test edilen: npm run lint (uyarı yok), npm run build (7 statik route prerender) — yeşil.
  • Adımlar:
    1. Netlify'da yeni site → repoya bağla.
    2. Build ayarları netlify.toml'dan otomatik alınır (Node 20, plugin aktif).
    3. İlk deploy sonrası /manifest.json, /sw.js ve DevTools > Application bölümünden Installability doğrulanmalı.
    4. Yeni sürümlerde public/sw.js içindeki VERSION sabitini artır → eski cache'ler temizlenir.
